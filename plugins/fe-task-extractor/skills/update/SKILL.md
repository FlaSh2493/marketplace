# Frontend Task Update (Sync to Jira)

사용자가 **마크다운 파일(.docs/task/ 하위 md 파일)을 직접 수정**한 후, 그 내용을 연결된 Jira 이슈에 그대로 반영(동기화)하는 스킬이다.

---

## 1. 대상 파일 확인

사용자가 파일명을 명시하지 않은 경우, `{프로젝트 루트}/.docs/task/` 폴더에서 동기화할 마크다운 파일을 선택하게 한다.

---

## 2. 분석 및 동기화 프로세스

### Step 1: 마크다운 내용 읽기 (Source of Truth)
대상 마크다운 파일을 읽어 각 작업의 제목, 설명, 의존성, API 정보, 그리고 연결된 Jira 이슈 키를 파악한다.

### Step 2: 변경 사항 식별
마크다운 내의 각 작업 항목(`FE-XX`)에 대해, 이미 생성된 `jira: PROJ-XXX` 키가 있는지 확인한다.

### Step 3: 메타데이터 업데이트
마크다운 상단의 `최근 업데이트: {YYYY-MM-DD HH:mm}` 필드를 현재 시간으로 갱신한다. (생성일은 유지한다)

### Step 4: Jira 업데이트 실행
Jira 키가 있는 각 작업에 대해, Atlassian MCP의 `jiraUpdateIssue`를 호출하여 마크다운의 최신 내용을 Jira 티켓에 반영한다.

**업데이트할 Jira 필드**:
- **Summary**: 마크다운의 작업 제목
- **Description**: 아래 형식으로 마크다운의 내용을 재구성하여 반영
```
{설명 1~2문장}

- 선행: {deps ←}
- 후행: {deps →}
- API: {api}
- UI 상태: {states}
```

---

## 3. 출력 및 승인

### Step 1: ✅ Gate 1: 동기화 확인/승인
마크다운 파일의 변경 사항 중 Jira에 반영될 항목들을 아래 표 형식으로 요약하여 보여주고 승인을 받는다.

---
### 📋 Jira 동기화 예정 목록

| Jira Key | 작업 제목 | 변경 내용 요약 |
| :--- | :--- | :--- |
| {PROJ-XXX} | {제목} | {제목/설명/API 등 변경 사항} |

**대상 파일**: {파일명}
---

사용자에게 다음과 같이 질문한다:
> "마크다운 파일의 변경 사항을 확인했습니다. 위 목록대로 Jira 티켓들을 업데이트할까요?"

### Step 2: 실행 및 보고
승인 시 `jiraUpdateIssue`를 호출하고 아래 형식으로 보고한다.

### ✅ Jira 동기화 완료

| Jira Key | 작업 제목 | 상태 |
| :--- | :--- | :--- |
| [{KEY}](URL) | {제목} | 업데이트 완료 |

---

## 4. 주의사항

- **마크다운 우선**: 이 스킬은 마크다운 파일의 내용을 Jira로 보내는 것이 주 목적이다. Jira의 내용을 마크다운으로 가져오는 반대 방향의 동기화는 수행하지 않는다.
- **새 작업 추가 건**: 마크다운에 새로 추가되었으나 `jira: 미생성`인 항목은 `extract` 스킬의 Jira 생성 프로세스를 안내하거나 별도로 생성 여부를 묻는다.
- **포맷 준수**: 마크다운 파일이 `FE-XX` 형식을 유지하고 있어야 정상적으로 파싱 및 동기화가 가능하다.
