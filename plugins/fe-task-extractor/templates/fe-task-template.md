# FE 작업 명세 표준 템플릿

> **이 파일은 모든 스킬(extract, fetch, init, update)이 마크다운 파일을 생성·수정할 때 반드시 따라야 하는 단일 포맷 정의(Single Source of Truth)입니다.**
> 포맷을 임의로 변형하거나, 필드를 추가·삭제·순서 변경하지 마십시오.

---

## 파일 구조

아래 코드 블록이 **정확한 출력 형식**입니다. `{...}` 플레이스홀더만 실제 값으로 교체하십시오.

````markdown
# FE 작업: {기능명}

- 생성일: {YYYY-MM-DD HH:mm}
- 최근 업데이트: {YYYY-MM-DD HH:mm}
- 출처: {extract | jira-fetch}
- jira: {에픽키 또는 프로젝트키 또는 "없음"}

---

> {한 줄 요약: 이 명세가 무엇인지}

- 총 작업: {N}개

---

## FE-01. {작업 제목}

{무엇을·왜 1~2문장. 핵심 동작은 → 화살표로 표현한다.}

- deps: {←선행 →후행 | 없음}
- api: {`METHOD /endpoint` 설명 | 없음}
- states: {상태 흐름 | 없음}
- jira: {PROJ-123 | 미생성}
- note: {확인 필요 사항 | 생략}

---

## FE-02. {작업 제목}

{설명}

- deps: {←선행 →후행 | 없음}
- api: {`METHOD /endpoint` 설명 | 없음}
- states: {상태 흐름 | 없음}
- jira: {PROJ-456 | 미생성}
- note: {확인 필요 사항 | 생략}

---
````

## 필드 규칙

### 헤더 영역

| 필드 | 규칙 | 예시 |
|------|------|------|
| `기능명` | 브랜치명 또는 기능명 그대로 사용 | `feature/login-ui` |
| `생성일` | 파일 최초 생성 시각. 이후 수정하지 않음 | `2026-03-25 15:30` |
| `최근 업데이트` | 파일을 수정할 때마다 현재 시각으로 갱신 | `2026-03-25 16:00` |
| `출처` | `extract` (문서에서 추출) 또는 `jira-fetch` (Jira에서 가져옴) | `jira-fetch` |
| `jira` | 에픽 키, 프로젝트 키, 또는 `없음` | `PROJ-100` |

### 요약 영역

| 필드 | 규칙 |
|------|------|
| 인용문(>) | 이 명세가 무엇인지 한 줄로 설명 |
| `총 작업` | 이 파일에 포함된 `FE-XX` 작업의 총 개수 |

### 작업 항목 (FE-XX)

| 필드 | 규칙 | 값이 없을 때 |
|------|------|-------------|
| `제목` | `## FE-{번호}. {제목}` 형식으로 H2 헤딩 | — (필수) |
| 설명 | 제목 아래 빈 줄 후 1~2문장 | — (필수) |
| `deps` | `←` 선행, `→` 후행. 여러 개는 쉼표 구분 | `없음` |
| `api` | 백틱으로 `METHOD /path` 감싸고, 뒤에 설명 | `없음` |
| `states` | `→` 순차, `·` 분기 | `없음` |
| `jira` | Jira 이슈 키. Jira 미생성 시 `미생성` | `미생성` |
| `note` | 원문 불명확 또는 확인 필요 시 기재 | **필드 자체를 생략** |

### 작업 항목 필드 순서 (반드시 준수)

```
- deps:
- api:
- states:
- jira:
- note:      ← 필요 시에만
```

이 순서를 변경하지 마십시오.

## 포맷 검증 체크리스트

파일을 저장하기 **직전에** 아래 항목을 모두 확인하십시오:

- [ ] 파일 첫 줄이 `# FE 작업: {기능명}`으로 시작하는가
- [ ] 헤더에 `생성일`, `최근 업데이트`, `출처`, `jira` 4개 메타 필드가 모두 있는가
- [ ] 요약 영역에 `총 작업: {N}개`가 있는가
- [ ] 각 작업이 `## FE-{번호}. {제목}` 형식인가 (번호는 01부터 순차)
- [ ] 각 작업의 필드 순서가 deps → api → states → jira → note(선택)인가
- [ ] 각 작업 사이에 `---` 구분선이 있는가
- [ ] 값이 없는 필드에 임의의 설명 대신 `없음`이 들어가 있는가
- [ ] `note` 필드는 확인이 필요한 경우에만 존재하는가 (불필요 시 필드 자체 생략)
