# PR 본문 템플릿

```markdown
## 개요

{변경사항 한 줄 요약. 왜 이 변경이 필요한지 포함.}

## 변경사항

- {주요 변경 1}
- {주요 변경 2}
- {주요 변경 3}

## 완료 조건

{task.md의 완료 조건 체크리스트 (있는 경우)}

- [ ] {체크리스트 항목}

## 테스트

- [ ] 로컬 테스트 완료
- [ ] lint / type-check / test 통과

{이슈 키가 있는 경우}
Closes #{이슈 키}
```

## 작성 지침

- 개요: "왜" 중심. "무엇을 바꿨는지"보다 "왜 이 변경이 필요한지" 서술
- 변경사항: 리뷰어가 읽기 쉽게 bullet point
- 스크린샷/GIF: UI 변경이 있으면 Before/After 첨부 권장
- 완료 조건: task.md 존재 시 그대로 포함, 없으면 섹션 생략
