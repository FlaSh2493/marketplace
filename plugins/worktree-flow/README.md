# Worktree Flow Plugin

Git worktree를 사용하여 여러 작업을 병렬로 수행할 수 있게 도와주는 플러그인입니다.

## 주요 기능

- **병렬 작업 환경 구축**: 여러 기능을 동시에 개발할 때 각 기능별로 격리된 워크트리를 생성합니다. 하나의 명령으로 여러 개의 워크트리를 동시에 생성할 수 있습니다.
- **WIP 자동 커밋**: Claude 응답이 끝날 때마다 현재 워크트리의 변경사항을 `WIP: ...` 메시지로 자동 커밋합니다. (복원용)
- **AI 기반 커밋 정리 및 머지**: 머지 전 AI가 자잘한 WIP 커밋들을 분석하여 **의미 있는 단위로 재구성**한 뒤 피처 브랜치에 병합합니다. 히스토리를 깔끔하게 관리할 수 있습니다.
- **태그 생성**: 머지 전 각 워크트리 상태를 `wt/{feature}/{task}` 태그로 남겨 언제든 복원 가능하게 합니다.

## 설치 및 초기 설정

플러그인을 설치한 후, **자동 WIP 커밋** 기능을 활성화하려면 프로젝트 설정을 초기화해야 합니다:

```bash
/worktree-flow:init-hooks
```

이 명령어는 `.claude/settings.json` 파일에 `Stop` 훅을 등록합니다.

## 사용 방법

### 1. 워크트리 생성
```
/worktree-flow:create
```
현재 브랜치와 관련된 명세서 디렉토리(`.docs/task/{브랜치명}/`) 내의 이슈별 개별 md 파일에서 정보를 추출하여 선택 목록을 제공합니다. 

### 2. 작업 시작 (Context 로드)
```
/worktree-flow:start
```
현재 워크트리의 정보를 분석하여 로컬 명세서(`.docs/task/{피처명}/{이슈키}.md`) 정보를 자동으로 로드하고 **기획(Plan) 모드**에 진입합니다.

### 3. 상태 확인
```
/worktree-flow:status
```
현재 생성된 모든 워크트리의 위치, 브랜치명, 변경사항 수, 커밋 상태를 확인합니다.

### 4. WIP 자동 커밋 제어
```bash
/worktree-flow:wip [on|off|status]
```
워크트리 내에서 자동 커밋 기능을 활성화하거나 비활성화합니다. `.wip-active` 파일 존재 여부로 상태를 판단합니다.

### 5. 머지 및 정리 (AI-Assisted)
```bash
/worktree-flow:merge {피처브랜치}
```
1. **분석**: AI가 워크트리 내의 WIP 커밋들을 분석하여 그룹화합니다.
2. **제안**: AI가 의미 있는 커밋 본문과 메시지를 제안합니다.
3. **정리**: 승인 시 AI가 `git reset --soft` 등으로 커밋을 깔끔하게 재구성합니다.
4. **병합**: 정리된 브랜치들을 피처 브랜치에 병합하고 워크트리를 삭제합니다.

## 디렉토리 구조
- `.worktrees/{task}/`: 워크트리 하위 경로 (또는 설정된 경로).
- `feat/{feature}--wt-{task}`: 워크트리 전용 임시 브랜치.
- `.wip-active`: 이 파일이 워크트리 루트에 있으면 해당 세션의 WIP 자동 커밋이 활성화됩니다.
