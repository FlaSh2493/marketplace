## 이슈 완료 시 (같은 spec 내 다음 이슈 존재)
1. 문서 자동 저장 (/jira-save 자동 호출)
   - plan.md 저장
   - {workDir}/jira-planner/specs/{spec}/issues/{이슈키}/ 에 저장
2. state.json 업데이트 + Jira 상태 변경
3. 이전 이슈 결과 요약 생성
   - 변경된 파일 목록
   - 새로 추가/수정된 인터페이스 (함수 시그니처, 타입 등)
   - 다음 이슈가 알아야 할 핵심 변경점만 (500토큰 이내)
4. 다음 이슈의 issues/{키}.md 읽기
5. 세션에 주입: 이전 결과 요약 + 다음 이슈 요구사항
6. Plan 모드 전환 안내

## Spec 완료 시 (멈춤)
1. state.json 전체 Done 처리
2. 최종 요약 출력 (완료 이슈, 문서 경로)
3. 다음 spec 안내만 출력 (자동 전환 안 함)
   - "준비되면 /jira-start {다음spec} 으로 시작하세요."
4. 세션 종료

## 승인 포인트 (사용자 입력이 필요한 곳)

| 시점 | 질문 | 예상 응답 |
|------|------|----------|
| Plan 완료 | "이 계획으로 진행할까요?" | "ㅇ" / 수정 요청 |
| 구현 완료 | "구현 결과를 확인해주세요" | "ㅇ" / 수정 요청 |

## 자동 처리 (사용자 입력 없이)
- 문서 저장
- Jira 상태 업데이트
- 이전 이슈 결과 요약 생성
- 다음 이슈 컨텍스트 로드
- state.json 업데이트
