# 프로젝트 커스텀 지침 참조 (필수)

본 스킬은 프로젝트별 특화된 지침을 행동에 반영하기 위해 다음 스크립트를 호출하여 `.autopilot-instructions/` 지침을 **무조건 참조**합니다.

## 지침 확인 및 반영 (STEP 0.5)

```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/load_custom_instructions.py {skill_name}
```

1. **필수 제약 사항**: 로드된 내용이 있다면 **본 스킬을 수행하는 동안 반드시 준수해야 하는 필수 제약 사항**으로 간주한다.
2. **기억 전이(Memory Handoff)**:
   - `plan` 스킬은 이후 단계(build 등)를 위한 지침까지 미리 읽어 `plan.md` 파일에 기록한다.
   - `build` 등 후속 스킬은 `plan.md`에 이미 지침이 기록되어 있다면 해당 내용을 그대로 따라 행동하며, 지침의 일관성을 유지한다.
3. **표준 절차 준수 및 왜곡 금지**:
   - `SKILL.md`에 명시된 표준 절차와 로직은 자의적으로 왜곡하지 않는다.
   - 커스텀 지침은 표준 절차의 틀 내에서 세부 규칙(스타일, 도구 등)을 주입하는 용도로만 사용한다.
   - 플러그인의 무결성과 안전 수칙은 커스텀 지침보다 우선한다.

지침 파일이 존재하지 않는 경우, 본 단계를 조용히 생략하고 기본 `SKILL.md` 지침에 따라 행동한다.
