# Worktree Status

**실행 주체: Main Session 전용**

## 사용법
`/autopilot:status`

## 실행 절차

STEP 1: 워크트리 목록 조회
  실행: `git worktree list --porcelain`
  메인 리포(첫 번째 항목) 제외한 워크트리 목록 파싱

  각 워크트리마다:
    - 브랜치: `git -C {경로} rev-parse --abbrev-ref HEAD`
    - 커밋 수: `git -C {경로} log --oneline | wc -l`
    - 마지막 커밋: `git -C {경로} log -1 --format="%ci"`

STEP 2: 결과 출력
  ```
  ┌──────────┬──────────────────────────┬────────┬────────────────────────┐
  │ 이슈     │ 브랜치                    │ 커밋수 │ 마지막 커밋             │
  ├──────────┼──────────────────────────┼────────┼────────────────────────┤
  │ PLAT-101 │ worktree-PLAT-101        │  3개   │ 2026-03-25 14:32:01    │
  └──────────┴──────────────────────────┴────────┴────────────────────────┘
  ```
  워크트리가 없으면: "활성 워크트리 없음"

  워크트리가 있으면 AskUserQuestion으로 다음 선택지 제시:
  ```
  다음 중 선택하세요:
  1. `/autopilot:plan {이슈키}` — 새 이슈 착수
  2. `/autopilot:work {이슈키} {요구사항}` — 기존 워크트리에 추가 작업
  3. `/autopilot:merge {피처브랜치}` — 워크트리를 피처 브랜치에 머지
  4. 추가 작업 계속
  ```
