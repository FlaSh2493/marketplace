# Worktree Status

**실행 주체: Main Session 전용**

## 사용법
`/autopilot:status`

## 실행 절차

STEP 1: 워크트리 목록 조회
  실행: `git worktree list --porcelain`
  메인 리포(첫 번째 항목) 제외한 워크트리 목록 파싱

  각 워크트리마다:
    - 브랜치: `git -C {경로} rev-parse --abbrev-ref HEAD`
    - 이슈 목록: `cat '{경로}/.autopilot'` → issues 배열 (없으면 "-")
    - 커밋 수: `git -C {경로} log --oneline | wc -l`
    - 마지막 커밋: `git -C {경로} log -1 --format="%ci"`

STEP 2: 결과 출력
  ```
  ┌──────────────────────┬──────────────┬────────┬────────────────────────┐
  │ 브랜치                │ 이슈          │ 커밋수 │ 마지막 커밋             │
  ├──────────────────────┼──────────────┼────────┼────────────────────────┤
  │ feat/filter          │ PLAT-101     │  3개   │ 2026-03-25 14:32:01    │
  │ feat/export          │ PLAT-102,103 │  1개   │ 2026-03-25 15:10:22    │
  └──────────────────────┴──────────────┴────────┴────────────────────────┘
  ```
  워크트리가 없으면: "활성 워크트리 없음"

  워크트리가 있으면 AskUserQuestion으로 다음 선택지 제시:
  ```
  다음 중 선택하세요:
  1. `/autopilot:plan {브랜치명} [이슈키]` — 워크트리 착수 또는 추가 작업
  2. `/autopilot:merge {피처브랜치}` — 워크트리를 피처 브랜치에 머지
  4. 추가 작업 계속
  ```
