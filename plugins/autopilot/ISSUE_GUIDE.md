# 이슈 명세 작성 가이드

autopilot의 플랜 기능을 최대한 활용하려면 이슈 명세에 아래 정보가 포함되어야 합니다.

---

## 명세 템플릿

```markdown
## 설명
무엇을 만드는지. 기획 의도와 사용자 가치를 포함한다.

예) 리포트 필터 기능 추가. 사용자가 날짜/카테고리/담당자별로 리포트를 필터링할 수 있어야 한다.

## 화면/디자인
- Figma 링크 또는 이미지 첨부
- 레이아웃 설명: "헤더 고정, 본문 스크롤, 하단 CTA 버튼 full-width"
- 간격/크기 힌트: "카드 간격 16px, 섹션 패딩 24px, 버튼 높이 48px"
- 상태별 UI: "필터 적용 시 칩으로 표시, 초기화 버튼 노출"

## 컴포넌트 힌트
- 사용 예상 공통 컴포넌트: Button, Select, DateRangePicker, Chip 등
- 디자인시스템 토큰: color.primary, spacing.md, radius.sm 등
- 모르면 비워도 됨 — Claude가 코드베이스에서 직접 탐색

## 영향 범위
- 수정 예상 파일/모듈: `src/pages/Report.tsx`, `src/api/report.ts`, `src/store/reportStore.ts`
- 공유 상태/API: `GET /api/reports?filter=...` — 다른 이슈와 겹칠 경우 명시
- 공통 타입 변경: `ReportFilter` 타입 추가 예정

## 완료 조건
- [ ] 날짜 범위 선택 시 리포트 리스트 즉시 반영
- [ ] 카테고리/담당자 복수 선택 가능
- [ ] 필터 초기화 버튼 동작
- [ ] 모바일 레이아웃 대응 (필터 접기/펼치기)
- [ ] 필터 상태 URL 쿼리 파라미터로 유지
```

---

## 섹션별 작성 요령

### 설명
- "무엇을" 중심으로 작성. "왜"(기획 의도)를 한 줄 추가하면 Claude가 더 적합한 구현을 선택한다.
- 나쁜 예: "필터 추가"
- 좋은 예: "리포트 필터 추가. 담당자별로 업무량을 추적하기 위해 필요."

### 화면/디자인
- **이미지나 Figma 링크가 있으면 Claude가 디자인시스템 컴포넌트를 자동 매핑한다.**
  - 버튼, 인풋, 카드 등 어떤 공통 컴포넌트를 쓸 수 있는지 식별
  - 간격 숫자 → 디자인시스템 spacing 토큰으로 변환
  - 컬러 → 디자인시스템 color 토큰으로 변환
- 이미지가 없으면 레이아웃 설명만 텍스트로 적어도 된다.

### 컴포넌트 힌트
- 알고 있는 컴포넌트명만 적으면 된다. 모르면 비워도 Claude가 탐색한다.
- 힌트가 있을수록 탐색 범위가 줄고 정확도가 올라간다.

### 영향 범위
- **다중 이슈를 한 워크트리에서 작업할 때 가장 중요한 섹션.**
- 같은 파일을 여러 이슈가 수정하면 Claude가 플랜 단계에서 충돌을 감지하고 처리 순서를 제안한다.
- API나 공통 타입 변경이 있으면 반드시 명시.

### 완료 조건
- 체크리스트 형태로 구체적으로 작성.
- 명확할수록 `/autopilot:check` 자동화 범위가 넓어지고 구현 누락이 줄어든다.

---

## 다중 이슈 작업 시 팁

하나의 워크트리에서 여러 이슈를 함께 작업할 때:

```bash
/autopilot:plan feat/sprint3-filter PLAT-101 PLAT-102
```

Claude가 두 명세를 합쳐 플랜을 수립한다. 이때 **영향 범위** 섹션이 핵심이다.

| 상황 | Claude의 동작 |
|------|--------------|
| PLAT-101과 PLAT-102가 동일 파일 수정 | 플랜에 충돌 명시, 통합 처리 또는 순서 제안 |
| PLAT-101이 만든 컴포넌트를 PLAT-102가 사용 | PLAT-101 먼저 구현하도록 순서 명시 |
| 두 이슈가 같은 API를 호출 | 파라미터 통합 여부 검토 후 플랜에 포함 |

## 이슈 문서 저장 위치

autopilot은 다음 경로에서 이슈 문서를 탐색한다:

```
{git_root}/tasks/{이슈키}/{이슈키}.md   ← 우선순위 1
{git_root}/docs/task/{이슈키}/{이슈키}.md    ← 우선순위 2
```

또는 `jira:` frontmatter로 매핑된 마크다운 파일도 인식한다:
```markdown
---
jira: PLAT-101
---
```
