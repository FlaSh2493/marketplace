# FE 작업 명세 표준 템플릿

> **이 파일은 extract 스킬이 마크다운 파일을 생성·수정할 때 반드시 따라야 하는 포맷 정의입니다.**
> jira-fetch 포맷은 `fe-jira-fetch-template.md`를 참조하십시오.
> 포맷을 임의로 변형하거나, 필드를 추가·삭제·순서 변경하지 마십시오.

---

## 파일 저장 규칙

### 디렉토리 구조
```
{프로젝트 루트}/.docs/task/{브랜치명}/
├── {JIRA-KEY}.md              ← 이슈별 개별 파일 (예: PROJ-101.md)
├── {JIRA-KEY}.md
└── assets/                    ← 이미지·첨부파일 저장
    ├── {JIRA-KEY}-{파일명}.png
    └── {JIRA-KEY}-{파일명}.pdf
```

### 파일명 규칙
- **Jira에서 가져온 이슈**: `{JIRA-KEY}.md` (예: `PROJ-101.md`)
- **extract로 생성한 이슈 (Jira 미생성)**: `FE-{번호}.md` (예: `FE-01.md`) → Jira 생성 후 `{JIRA-KEY}.md`로 리네임
- **브랜치명의 `/`는 폴더 구조로 변환**: `feature/login-ui` → `.docs/task/feature/login-ui/`

## 단일 이슈 파일 구조

기획서·PRD에서 추출한 태스크(`출처: extract`) 포맷.

````markdown
# {JIRA-KEY}: {작업 제목}

- jira: {JIRA-KEY | 미생성}
- 상태: {Jira Status | 신규}
- 담당자: {Assignee}
- 생성일: {YYYY-MM-DD HH:mm}
- 최근 업데이트: {YYYY-MM-DD HH:mm}
- 출처: extract

---

## 목적

{왜 이 작업이 필요한가. 1~2줄}

---

## 요구사항

- [ ] 요건 1
- [ ] 요건 2

---

## 완료 기준

- [ ] 사용자가 X를 하면 Y가 된다

---

## 선행 이슈

- SPT-XXXX (없으면 생략)

---

## 첨부 이미지

{이미지가 있으면 아래 형식으로, 없으면 이 섹션 자체를 생략한다.}

![{이미지 설명}](./assets/{JIRA-KEY}-{파일명}.png)

---

## 첨부 파일

{이미지 외 첨부파일이 있으면 아래 형식으로, 없으면 이 섹션 자체를 생략한다.}

- [{파일명}]({다운로드 URL 또는 로컬 경로})

---

## 댓글

{댓글이 있으면 아래 형식으로 시간순 기재, 없으면 이 섹션 자체를 생략한다.}

> **@{작성자}** ({YYYY-MM-DD HH:mm})
> {댓글 내용}

---
````

## 필드 규칙

### 헤더 영역

| 필드 | 규칙 | 예시 |
|------|------|------|
| `JIRA-KEY` | 제목과 `jira` 필드에 동일하게 사용 | `PROJ-101` |
| `작업 제목` | Jira Summary 그대로. 임의 수정 금지 | `로그인 폼 UI 구현` |
| `상태` | Jira Status 그대로 | `To Do`, `In Progress` |
| `담당자` | Jira Assignee | `@홍길동` |
| `생성일` | Jira created date. 이후 수정하지 않음 | `2026-03-25 15:30` |
| `최근 업데이트` | 파일을 수정할 때마다 현재 시각으로 갱신 | `2026-03-25 16:00` |
| `출처` | `extract` (문서에서 추출) 또는 `jira-fetch` (Jira에서 가져옴) | `jira-fetch` |

### 헤더 필드 순서 (반드시 준수)

```
- jira:
- 상태:
- 담당자:
- 생성일:
- 최근 업데이트:
- 출처:
```

### 본문 섹션

| 섹션 | 규칙 |
|------|------|
| `## 목적` | 왜 이 작업이 필요한가. 1~2줄. 요약 금지 |
| `## 요구사항` | 체크박스 목록. 무엇을 만드는가 |
| `## 완료 기준` | 체크박스 목록. 언제 완료로 보는가 (사용자 시나리오 기반) |
| `## 선행 이슈` | 이슈키 목록. 없으면 "없음" 또는 섹션 유지하되 내용 생략 |

### 선택적 섹션 (없으면 섹션 자체 생략)

| 섹션 | 포함 조건 |
|------|-----------|
| `첨부 이미지` | 이미지 첨부파일이 1개 이상 존재할 때 |
| `첨부 파일` | 이미지 외 첨부파일이 1개 이상 존재할 때 |
| `댓글` | 댓글이 1개 이상 존재할 때 |

### 댓글 규칙

| 규칙 | 설명 |
|------|------|
| **시간순 정렬** | 오래된 댓글이 위, 최신 댓글이 아래 |
| **전체 기재** | 댓글을 요약하지 않고 원본 그대로 기재 |
| **인용 블록** | 각 댓글은 `>` 인용 블록으로 감싼다 |
| **빈 줄 구분** | 댓글 간에 빈 줄 1개로 구분 |

## 포맷 검증 체크리스트

파일을 저장하기 **직전에** 아래 항목을 모두 확인하십시오:

- [ ] 파일명이 `{JIRA-KEY}.md` 또는 `FE-{번호}.md` 형식인가
- [ ] 파일 첫 줄이 `# {JIRA-KEY}: {작업 제목}` 형식인가
- [ ] 헤더에 `jira`, `상태`, `담당자`, `생성일`, `최근 업데이트`, `출처` 6개 필드가 모두 있는가
- [ ] 헤더 필드 순서가 규칙대로인가
- [ ] `## 목적`, `## 요구사항`, `## 완료 기준`, `## 선행 이슈` 섹션이 존재하는가
- [ ] 선택적 섹션(첨부 이미지, 첨부 파일, 댓글)은 내용이 있을 때만 존재하는가
- [ ] 각 섹션 사이에 `---` 구분선이 있는가
- [ ] 이미지 참조 경로가 `./assets/{JIRA-KEY}-{파일명}` 형식인가
